**Purpose**

This repository is a homework starter for building a minimal terminal agent called nanocli that uses the OpenAI Responses API’s structured tool-calling interface instead of parsing bash blocks out of plain assistant text. The assignment expects the student to finish the missing tool and loop logic in tool.py and core.py, then verify it with the unit tests in test_tool.py and test_core.py.

At a high level, the project has three pieces:
1. A thin terminal UI in cli.py that prompts the user, prints assistant/tool output, and asks for approval before side-effecting tools.
2. A session and agent loop in core.py that sends conversation state to the OpenAI Responses API and handles returned tool calls.
3. A local tool layer in tool.py that defines and executes four tools: bash, read_file, list_dir, and edit_file.

**How To Run**

The intended workflow is described in README.md and hw2.md:

- Install dependencies with uv.
- Run tests with `uv run pytest`.
- Run lint/hooks with `prek run --all-files`.
- Start the mock server with `uv run nanocli-mock`.
- Start the CLI with `uv run nanocli --model <model-name>`.

Two console entry points are defined in pyproject.toml:
- `nanocli` → launches the interactive CLI
- `nanocli-mock` → launches the local mock Responses API server from mock_server.py

Important caveat: this is intentionally incomplete starter code. The README explicitly says tests may fail until the `STUDENT TODO` sections are implemented.
