The repository is a **minimal “nano” command‑line agent** (`nanocli`) whose sole job is to
demonstrate the kind of tool‑using loop you build on top of the new
[Responses API](https://api.openai.com/v1/responses).  
It’s the homework for CS598‑LMZ (Spring 2026) and is **mostly scaffolding and
tests** – the student fills in a couple of `# STUDENT TODO` spots to make the
agent actually work.

---

## 🔍 What the project does

* Defines a very small data model in core.py – `Message`,
  `Reasoning`, `ToolCallInput/Output`, `Session`, etc.  
* Implements `run_turn()` (to be completed by students) that:
  1. sends the current session context and instructions to `client.responses.create()`
  2. streams back model output (assistant messages, reasoning items and
     `function_call` events)
  3. for every tool request it yields a `ToolCallInput`, optionally waits for
     user approval, executes the tool and yields a `ToolCallOutput`, then feeds
     the result back into the next request.
* Provides four local tools that the model can call:
  * `bash` – run any shell command in `Session.cwd`
  * `read_file` – read lines of a UTF‑8 file with cat‑`-n`‑style numbering
  * `list_dir` – return a sorted directory listing (`dirs/` suffix)
  * `edit_file` – perform a single search‑and‑replace (error on ≠1 match)
* Wraps the loop in a tiny terminal UI (cli.py) that prints
  coloured labels, requests confirmation (unless `--yolo`), etc.
* Contains a mock Responses API server in mock_server.py
  used by the unit tests to simulate model behaviour and trigger tool flows.

## 🚀 How to run

1. **Set up the environment** – the project uses [`uv`](https://github.com/astral-sh/uv) to
   manage a Python environment and dependencies.  
   ```bash
   uv run python -m pip install -r requirements.txt    # solved automatically
   uv run pytest                                       # run the unit tests
   ```
   *Pre‑commit* checks are handled by [`prek`](https://github.com/j178/prek); install
   with `uv tool install prek` and run `prek run --all-files`.

2. **Interactive CLI** – once your implementation passes the tests you can
   exercise the agent on a real model:
   ```bash
   uv run python -m nanocli.cli --model <model-name> [--yolo] \
       [--instructions-path path/to/configs/tool_use_instructions.txt]
   ```
   Type messages at the `User>` prompt; the agent will call tools, print results
   and keep the conversation state in memory.

3. **Tests** – the tests directory contains exhaustive unit tests for:
   * the tool implementations (test_tool.py)
   * the `run_turn` loop and approval logic (test_core.py)
   Running `uv run pytest` exercises these; initially several tests fail at
   `STUDENT TODO` locations in core.py and tool.py.

### Tools exposed to the model

Each tool is registered in `nanocli.tool` and declares a strict JSON schema
(`FunctionToolParam`) so that the Responses API can validate and generate
correct arguments. `tool_specs()` collects them for the API call.
