The codebase you opened is **not a single application** but a collection of homework assignments for the
“CS 769 – Advanced NLP” course.  
Each top‑level directory (assignment1, assignment2, … , project) is essentially a
self‑contained mini‑project with its own README, data and helper scripts.  
Below is a high‑level tour of what the repository is intended to teach, how you
exercise each assignment, and what tools/APIs it relies on.

---

## 🔍 Overall intent

* **Assignment 1** – build a simple text classifier (Deep Averaging Network) in
  PyTorch; learn basic model definition, parameter init, batching, and embedding
  loading.
* **Assignment 2** – implement a minimalist BERT (“minbert”) from scratch and
  pre‑train / fine‑tune it for sentence classification; exposes you to
  transformers‑style code.
* **Assignment 3** – course‑project **proposal**; no code to run, just survey and
  write‑up.
* **Assignment 4** – implement beam‑search helpers for a toy encoder–decoder;
  the goal is to understand `topK`, hidden‑state selection, and sequence
  extraction.
* **project/** – guidelines for the end‑of‑semester course project (report,
  submission, grading criteria).

Each assignment supplies a skeleton implementation, tests/sanity checks, and
a run_exp.sh script demonstrating the exact commands the instructors will
use to grade your work.

---

## ▶️ How to run

For the code‑heavy assignments (1, 2, 4) the workflow is basically the same:

1. **Set up the environment**  
   ‑– often by running the provided `setup.sh` (assignment 2) or leaving `setup.py`
   for any downloads (assignment 1).  
   These install the only third‑party dependencies: `torch` (PyTorch) and,
   for assignment 1, `numpy`.

2. **Execute the experiment script**  
   e.g. from assignment1:

   ```bash
   # example from run_exp.sh
   CAMPUSID="9xx1234567"
   python main.py \
       --train data/sst-train.txt \
       --dev data/sst-dev.txt \
       --test data/sst-test.txt \
       --dev_out $CAMPUSID/sst-dev-output.txt \
       --test_out $CAMPUSID/sst-test-output.txt
   ```

   or from assignment2:

   ```bash
   python classifier.py --use_gpu --option finetune --lr 1e-5 \
       --train data/sst-train.txt --dev data/sst-dev.txt --test data/sst-test.txt \
       --dev_out $CAMPUSID/sst-dev-output.txt --test_out $CAMPUSID/sst-test-output.txt
   ```

   Assignment 4 has a simple sanity test:

   ```bash
   python test_beam_search.py
   ```

3. **(Optional) prepare submission** – each run_exp.sh shows how to collect
   outputs and source files into a `CAMPUSID/` directory and zip it with
   `prepare_submit.py`.

## 🧱 Code structure & APIs

* **PyTorch** is the core deep‑learning library used across assignments.
  - models are defined as `torch.nn.Module` subclasses (`model.py`,
    `bert.py`, etc.).
  - training loops use `torch.optim` optimizers and standard tensor operations.
* **NumPy** appears only in assignment 1 (embedding loading, initialization).
* No other third‑party libraries are permitted – e.g. you may **not** import
  `transformers` or `sklearn`.
* CLI argument parsing is handled with Python’s `argparse` in the `main.py` /
  `classifier.py` scripts.
* Utility files:
  - `vocab.py` constructs vocabularies from tokenized text.
  - `tokenizer.py`, `utils.py`, `optimizer.py` in assignment 2 support BERT.
  - beam_search.py contains three functions (`topK`, `select_hiddens_by_beams`,
    `extract_sequences`) that the student must implement.
* Sanity‑check data and pretrained models are included (sanity_check.pt,
  `sanity_check.data`).
* Shell scripts (run_exp.sh, `setup.sh`) demonstrate reproducible commands and
  aid submission preparation.
