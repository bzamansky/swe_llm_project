**What It Is**

This repository is a starter homework project for building a minimal CLI coding agent called `nanocli`. The intended behavior is a bash-only ReAct-style loop: the model replies through the OpenAI Responses API, the core looks for exactly one fenced `bash` block, asks for approval unless `--yolo` is set, runs that shell command, feeds the structured output back into conversation context, and repeats until the model stops emitting bash blocks. The assignment spec is in hw1.md, the unfinished core is in core.py, the terminal UI is in cli.py, and the behavior contract is encoded in test_core.py.

As checked, this checkout is still scaffold code rather than a finished agent: the three required functions in core.py still raise `NotImplementedError`, so the repo is intended to be completed by the student before tests pass.

**How To Run It**

Environment and workflow are documented in README.md and hw1.md. The repo expects Python 3.12 and uses `uv` for environment management.

Typical commands are:

```bash
uv run pytest
prek run --all-files
uv run nanocli --model <model>
uv run nanocli-mock
```

For local end-to-end testing, the repo includes a mock Responses API server in mock_server.py. The intended flow is:

```bash
export OPENAI_BASE_URL=http://localhost:8000/v1
export OPENAI_API_KEY=dummy
uv run nanocli-mock
uv run nanocli --model mock --yolo
```

The CLI supports `--model`, `--yolo`, `--instructions-path`, `--instructions-as-system`, and `--reasoning_effort`; those are defined in cli.py. The default instructions file is bash_only_instructions.txt, which tells the model to either return normal text or exactly one fenced bash block.

**APIs, Commands, Libraries**

The main external API is the OpenAI Responses API, used via the Python `openai` SDK in core.py. The code constructs `client.responses.create(...)` requests with streaming enabled, `store=False`, `include=["reasoning.encrypted_content"]`, optional `instructions`, optional system-message fallback, and optional reasoning settings. The mock server exposes a compatible local endpoint at `POST /v1/responses` plus `GET /health` and a shutdown endpoint in mock_server.py.

The core shell command being modeled is plain bash execution through Python `subprocess`; the assignment expects commands like `pwd`, `ls`, `whoami`, and failing shell commands to be captured with `stdout`, `stderr`, and exit code, as shown by test_core.py. The project entry points are declared in pyproject.toml: `nanocli` for the agent and `nanocli-mock` for the mock server.

Libraries and tools used in the repo:

- Runtime: `openai` in pyproject.toml
- Dev/test: `flask`, `pytest`, `ipython` in pyproject.toml
- Lint/format/type check: `ruff-format`, `ruff --fix`, and `pyright` via `prek`, configured in .pre-commit-config.yaml and pyproject.toml
- Environment management: `uv`, documented in README.md
