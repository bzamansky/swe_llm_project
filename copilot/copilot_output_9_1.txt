**What The Project Is Intended To Do**
The assignment is to build a small HTTP/1.1 server from raw sockets, with a simple router and several incremental web features. The spec in README.md and the handler registrations in main.py show these goals:

- Parse raw HTTP requests and write raw HTTP responses.
- Serve static files from the local data folder.
- Accept JSON POST data and return the last posted value.
- Return a 302 redirect.
- Implement cookie-based authentication.
- Implement session-based authentication with a random session key.
- Expose browser test pages for the auth tasks.

The concrete routes are registered in main.py:

- / with the default handler, which returns 404.
- /data for GET and HEAD static file serving.
- /post for GET, HEAD, POST JSON storage/retrieval.
- /redirect for GET and HEAD 302 redirect.
- /api/login and /api/getimage for cookie auth.
- /apiv2/login and /apiv2/getimage for session auth.
- /api/test and /apiv2/test for browser testing.

The server itself binds to 127.0.0.1:8080, checks that the Host header matches exactly, and always sends Connection: close, as shown in framework.py and config.py.
