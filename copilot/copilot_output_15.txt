**Purpose**

This repository is a handout/starter project for a Python assignment: build a command-line housing priority calculator that assigns a score to a student based on answers to a few questions.

The intended workflow, based on README.md, is:

- Design a scoring policy for housing priority.
- Write tests first.
- Implement the code after the tests describe the behavior.
- Run tests and coverage.
- Document the scoring rationale in Summary.md.

The program itself is split into three roles:

- question_asker.py: interactive input collection
- priority_calculator.py: scoring logic
- hw1.py: orchestration of asking questions and computing the final score

At the moment, the source files are skeletons with docstrings and pass statements, so this repo is describing the assignment more than providing a finished app.

**What The Project Is Intended To Do**

The assignment asks the student to build an interactive CLI program that:

- asks for class year
- asks graduation status, but only for seniors
- asks how many credits have been earned
- asks exactly two extra yes/no questions chosen by the student
- computes a housing priority score from those inputs
- prints the result

The key design intent is object-oriented separation of concerns:

- question_asker.py should validate and normalize user input
- priority_calculator.py should translate inputs into point values
- hw1.py should coordinate the flow between the other two classes

The tests in test_question_asker.py, impl_priority_calculator.py, impl_additional_questions.py, and impl_hw1.py show the expected behaviors and how the classes are supposed to interact.
