**Project Goal**

This repository is a course handout for a Python assignment: a housing priority score calculator. The student is expected to design and implement a small console program that asks a student questions, computes a housing-priority score, and uses tests-first development.

The intended architecture is split across three source files:

- README.md: assignment spec and workflow
- question_asker.py: user-input collection
- priority_calculator.py: scoring logic
- hw1.py: orchestration / program entry point

The repo as checked in is mostly a scaffold:
- the core methods in question_asker.py, priority_calculator.py, and hw1.py are still `pass`
- the tests in test_question_asker.py, impl_priority_calculator.py, impl_additional_questions.py, and impl_hw1.py are also partially unfinished
- Summary.md is a template the student is supposed to fill in with their chosen scoring system and reflection

**How It Is Intended To Work**

The design described by the handout is:

1. `HousingQuestionAsker` prompts for:
   - class year
   - graduation status, but only for seniors
   - credits earned
   - exactly two additional yes/no questions

2. `HousingPriorityCalculator` converts those answers into points:
   - class year points
   - graduation bonus
   - credit-based points
   - points from the two additional questions
   - total combined score

3. `calculate_score()` in hw1.py coordinates both classes:
   - instantiate asker and calculator
   - gather inputs
   - conditionally ask graduation status only for seniors
   - compute and return total score

4. `main()` prints a simple CLI header/footer and runs the flow

So the assignment is less “build a finished app from this repo” and more “complete the scaffold by writing tests first, then implementing the program.”
