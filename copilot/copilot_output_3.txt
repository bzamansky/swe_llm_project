The code in this workspace is **the skeleton for Homework 3 of CS598‑LMZ**.  
It builds on the earlier “nanocli” agent from HW 2 and adds a new layer of **agent skills**.

---

## 🔍 What the project does

1. **Implements a tiny command‑line agent (`nanocli`)** that
   - maintains a chat-like `Session` with a model,
   - sends context to the **OpenAI Responses API** (via the Python `openai` package),
   - and drives the ReAct‑style tool‑calling loop.
2. **Provides a suite of local tools** (`bash`, `read_file`, `list_dir`,
   `edit_file`) whose schemas are registered and exposed to the model.
3. **Adds “skills” support**:
   - skills are directories under skills with a `SKILL.md` containing
     YAML front‑matter;
   - the agent *implicitly* advertises available skills in the initial
     user‑instructions prompt;
   - the agent can *explicitly* inject a skill on a per‑turn basis when
     the user prefixes the message with `$skill-name`.
4. The homework beginnings include four `STUDENT TODO` markers where
   discovery, rendering, explicit resolution and turn‑loop integration
   must be implemented.

---

## 🚀 How to run

1. **Environment**  
   Uses [`uv`](https://github.com/astral-sh/uv) to manage a Python
   virtual environment.  
   Install with `uv run` commands; e.g. `uv run python`, `uv run pytest`.

2. **Dependencies & tooling**  
   - Python packages are declared in pyproject.toml (OpenAI client,
     `pytest`, …).
   - [`prek`](https://github.com/j178/prek) is the pre‑commit replacement;
     run it with `prek run --all-files`.

3. **Tests**  
   ```bash
   uv run pytest -v          # exercises core loop, skills, tools, etc.
   ```
   Failing tests indicate the missing TODOs.

4. **Mock server and CLI**  
   - `uv run nanocli-mock` starts a local mock Responses server.
   - `uv run nanocli --model nothing` launches the interactive agent.

## 🧩 APIs, commands & libraries used

| Purpose | Library / command |
|---------|-------------------|
| LLM interaction | `openai` Python package, **Responses API** (chat & tool calls) |
| Skill files | Local filesystem (`pathlib.Path`, `read_text`) |
| Subprocess execution | `subprocess.run` (for `bash` tool) |
| JSON parsing/serialization | `json` module |
| Unit testing | `pytest` and custom `MockResponsesServer` |
| Data modeling | `dataclasses` |
| CLI environment | [`uv`](Rust‑based env mgr), `prek` for linting |
| File I/O tools | implemented in `nanocli/tool.py` (`read_file`,
`list_dir`, `edit_file`) |
| Regex for tests | `re` (to detect `<skill>` blocks) |

The agent loop lives in core.py; skill helpers are in
skill.py; tools and registry are in tool.py.
