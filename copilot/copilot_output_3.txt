**Project purpose**

This repository is a homework starter for a minimal terminal coding agent called NanoCLI. The assignment is to extend an HW2-style tool-calling agent with skill discovery and skill injection, so the agent can preload available skills and inject a selected skill into a turn before continuing the normal tool loop. The assignment brief is in hw3.md, and the runtime shape is in core.py, skill.py, and tool.py.

As checked in, this is still a starter repo, not a finished app: several functions in core.py, skill.py, and tool.py are intentionally left as TODOs and are expected to be implemented by the student.

**What it is intended to do**

At a high level, NanoCLI is meant to:

1. Start an interactive CLI session from cli.py.
2. Send the conversation to the OpenAI Responses API from core.py.
3. Let the model request local tools.
4. Execute those local tools on the machine:
   bash, read_file, list_dir, edit_file
5. Feed tool outputs back into the model until the model stops requesting tools.
6. Add a new HW3 layer for skills:
   discover local skills under the skills directory,
   preload skill metadata into the prompt,
   detect explicit mentions like `$alpha-skill`,
   inject the full SKILL.md content for that turn.

The included sample skills in SKILL.md and SKILL.md show the intended format and use case.

**How it should be run**

The repo expects a Python 3.12 environment managed with `uv`, per README.md and pyproject.toml.

Main ways to run it:

1. Start the CLI:
```bash
uv run nanocli --model <model-name>
```

2. Start the mock Responses API server:
```bash
uv run nanocli-mock
```

3. Run tests:
```bash
uv run pytest
```

4. Run linting/checks:
```bash
prek run --all-files
```

5. Optional REPL work:
```bash
uv run python
uv run ipython
```

Practical note: the real CLI in cli.py constructs an `OpenAI()` client, so real usage will need the normal OpenAI Python client environment configuration, typically an API key. For tests, the repo uses its own mock server in mock_server.py.

**APIs used**

The main external API is the OpenAI Responses API through the official Python SDK in core.py.

Specifically, the code uses:

1. `OpenAI`
2. `client.responses.create(...)`
3. Streaming responses with `stream=True`
4. Tool registration via the Responses tool schema
5. Response item types such as:
   `message`
   `reasoning`
   `function_call`
   `function_call_output`

The code also uses OpenAI SDK response typing helpers from `openai.types.responses`, including typed message, reasoning, and function-call payloads.

A second API surface is the local mock HTTP server in mock_server.py, which uses Flask plus server-sent events to emulate the Responses API for testing.
