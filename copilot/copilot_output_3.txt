**Purpose**

This repo is a homework starter for a minimal Python coding agent called `nanocli`. The assignment is to extend an HW2-style tool-calling loop with “skills”: discover local SKILL.md files, preload their metadata into prompt context, and inject a full skill block when the user explicitly prefixes a turn with something like `$alpha-skill`. That intent is spelled out in hw3.md and reflected in the unfinished hooks in core.py and skill.py.

As checked in, it is not a finished app yet. The docs explicitly say tests will initially fail until the student fills the TODOs, and the code still has `NotImplementedError` placeholders in the core loop, skill handling, and HW2 tool implementations. See README.md, hw3.md, core.py, skill.py, and tool.py.

**How To Run**

The intended workflow is:

- Install dependencies with `uv`, then run everything through `uv run`, per README.md.
- Start the interactive CLI with `uv run nanocli --model <model>`. The CLI entry point is cli.py, and the console script is declared in pyproject.toml.
- Start the mock OpenAI-compatible server with `uv run nanocli-mock`, from mock_server.py.
- Run tests with `uv run pytest`, from README.md.
- Run checks with `prek run --all-files`, from README.md.
- Inside the CLI, `/debug` dumps the current instructions and assembled context, from cli.py.

The main CLI flags are `--model`, `--yolo`, `--instructions-path`, `--instructions-as-system`, and `--reasoning_effort`, defined in cli.py. In real use, the code constructs a default `OpenAI()` client in core.py, so it expects normal OpenAI SDK configuration such as an API key or a compatible `base_url`.

**APIs, Commands, Libraries**

The core external API is the OpenAI Python SDK. `Session.stream_response_final()` calls the Responses API with streaming, tool definitions, and optional reasoning effort via `client.responses.create(...)` in core.py. It also uses typed response objects from `openai.types.responses` in core.py and strict function-tool schemas in tool.py.

The agent exposes four local tools to the model, all defined in tool.py:
- `bash` for shell commands, at tool.py
- `read_file`, at tool.py
- `list_dir`, at tool.py
- `edit_file`, at tool.py

The skill system lives in skill.py, with example skills in SKILL.md and SKILL.md. The default agent instructions come from tool_use_instructions.txt.

Declared libraries are:
- Runtime: `openai>=2.2.0` in pyproject.toml
- Dev/test: `flask`, `ipython`, `pytest` in pyproject.toml
- Build backend: `uv_build` in pyproject.toml
