**Project**
This repository is a three-stage research pipeline for estimating food volume from a single RGB image, described in README.md.

1. Food detection/classification: fine-tunes Inception-ResNet-V2 on Food-101 or Recipe-1M to classify food images. The implementation is in model.py and evaluate.py.
2. Segmentation: uses a RefineNet-style semantic segmentation model with a ResNet-101 backbone, starting from Pascal VOC pretraining and then fine-tuning on food masks. The model code is in model.py and the train/demo entry points are in multi_gpu_train.py and demo.py.
3. Volume estimation: predicts a depth map from one image, then combines that depth with a LabelMe-style segmentation JSON and a fixed plate-size prior to estimate per-food volume. The entry point is demo.py, and the geometry is in volume.py.

There is no single end-to-end launcher. The repo is organized as three mostly separate subprojects.

**How To Run**
The intended commands are the ones in README.md, with each stage run from its own folder.

1. Food detection, from Food Detection/src:
   python3 convert_dataset.py --tfrecord_filename=foods --dataset_dir=../dataset/food-101/images
   python model.py --model=pretrained-inception-resnet-v2 --dataset=../dataset/FOOD101/images
   python evaluate.py --model /path/to/checkpoint_dir --dataset /path/to/dataset_dir

2. Segmentation, from Segmentation Mask:
   python convert_pascal_voc_to_tfrecords.py
   python RefineNet/multi_gpu_train.py
   python RefineNet/demo.py

3. Volume estimation, from Volume Estimation:
   python demo.py
   python demo.py --img /path/to/image.jpg --json /path/to/labelme.json --output /path/to/output

Required external assets are not bundled. You need Food-101 or similar data, segmentation TFRecords, pretrained checkpoints for RefineNet, and the depth model file loaded in demo.py. The demo also expects a LabelMe-style JSON and uses a hard-coded plate prior in volume.py.

**APIs, Commands, Libraries**
The main framework APIs are legacy deep-learning APIs rather than web or service APIs.

1. TensorFlow 1.x APIs:
   tf.app.flags for CLI flags in convert_dataset.py and the segmentation scripts.

2. PyTorch 0.4-era APIs:

3. Supporting libraries used across the repo:
   NumPy
