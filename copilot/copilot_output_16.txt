This repository is a CS61C coursework workspace, not a single deployable app. The starter materials live under CS61C-Assignment, and the matching reference solutions live under sol. The high-level intent is to practice computer architecture and systems topics across C, RISC-V, digital logic, parallelism, networking, distributed data processing, and a Python-backed native extension. The top-level summary is in README.md, but many per-project READMEs are placeholders, so the real run behavior comes from the build files and test scripts such as Makefile, Makefile, test_runner.py, and Makefile.

How it should be run depends on which assignment you are in. For general setup, the repo expects a C toolchain such as gcc or clang plus make, and it explicitly tells you to refresh Venus and Logisim with the script in tutorial.md and download_tools.sh:
bash tools/download_tools.sh

The main runnable entrypoints are:

1. C labs and proj1
make in directories like lab02, lab07, lab09, lab10, and proj1
Common targets visible in the Makefiles include:
make
make ex2
make ex3
make simd
make gameoflife
make imageloader
make steganography
make test
Several directories also support valgrind-based checks through explicit targets in the Makefiles.

2. RISC-V assembly project
proj2 is a Venus-based RISC-V project. The unit-test framework in framework.py shells out to Java and runs tools/venus.jar directly. The repo does not provide a wrapper Makefile here; the intended path is Python unittest-style execution against unittests.py, with Venus present in the project tools directory. There is also a matrix-format conversion helper in convert.py.

3. CPU / Logisim project
proj3 is a Logisim CPU project. The explicit test runner is test_runner.py, which runs:
python3 test_runner.py
python3 test_runner.py part_a
python3 test_runner.py part_b group_name test_file.circ
Internally it executes Java against logisim-evolution.jar with the tty table mode. The .circ files under cpu, harnesses, and tests are intended to be opened or simulated with Logisim-evolution.

4. Circuit labs
The digital-logic labs use .circ files and Logisim. Lab 5 also includes an explicit Python test harness in test.py, which you run as:
python3 testing/test.py

5. Parallel / networking lab
lab10 builds OpenMP demos and two HTTP server variants. The Makefile builds v_add, dotp, hello, server_basic, and server_process. The server CLI is defined in server.c and accepts flags like:
./server_basic --files ./files --port 8000
./server_process --files ./files --port 8000
There is also a --dotp-size flag.

6. Spark / Hadoop lab
lab11 is a MapReduce / Spark lab. The Makefile includes Hadoop sequence-file generation:
make generate-input myinput=some_text_file
The Python scripts in that directory create their own local SparkContext, so they can be run directly as:
python3 wordCount.py seqFiles
python3 createIndices.py seqFiles
python3 perWordDocumentCount.py seqFiles
python3 mostPopular.py seqFiles

7. Python C-extension project
proj4 is a native Python matrix library called numc. The intended workflow in the Makefile is:
make install
make test
make uninstall
The install path uses setup.py to build and install a Python extension, and the C test target builds a local binary from mat_test.c and matrix.c.
