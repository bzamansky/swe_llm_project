The repository you’ve opened is a small homework assignment (HW3) for building and training
**neural‑net models on CIFAR‑10 cat/dog data**.

## 📁 Project structure

```
assignment.py         # driver script (load data, train/test models, visualise)
base_model.py         # abstract base class with loss/accuracy helpers
cnn.py                # convolutional‑network subclass of CifarModel
mlp.py                # fully‑connected (MLP) subclass of CifarModel
manual_convolution.py # hand‑rolled conv2d layer and conv helper
preprocess.py         # CIFAR‑10 loading, filtering and batching utilities
local_test.py         # a collection of sanity/unit tests for the conv layer
```

All code is written in Python and assumes a single workspace folder.

---

## 🎯 Intended functionality

1. **Data handling** – `preprocess.get_data` reads a pickled CIFAR batch,
   filters it to the two classes requested (`cat` vs `dog`), normalises the images
   and one‑hots the labels.  `get_next_batch` slices batched data.

2. **Model classes** – `base_model.CifarModel` defines the interface and helpers
   (`loss`, `accuracy`) which both `CNN` and `MLP` inherit from.  
   Students must implement:
   * a forward pass in `CNN.call` / `MLP.call`
   * the loss/accuracy functions in `base_model`
   * network layers & hyper‑parameters in the constructors
   * the training (`train`) and evaluation (`test`) loops in assignment.py.

3. **Manual convolution** – `ManualConv2d` is a custom `tf.keras.layers.Layer`
   whose `call` method should perform 2‑D convolution **without** using
   `tf.nn.conv2d`.  The accompanying local_test.py script contains
   multiple tests comparing the custom layer to `keras.layers.Conv2D`.

4. **Training driver** – `assignment.main` is the entry point for loading the
   CIFAR files, instantiating either the `CNN` or `MLP` model, running a number
   of epochs, and reporting accuracy & loss.  There are helper visualisation
   functions for plotting the loss curve and correct/incorrect examples.

The comments make it clear that the goal is to achieve **≥ 70 % test accuracy** on
the cat/dog subset with a TensorFlow model trained on CPU only.

## 🛠 How to run

* **Local testing of conv layer** – from the repo root:

```bash
python local_test.py
```

Uncomment desired tests to exercise the `ManualConv2d` implementation.

* **Training & evaluation** – edit assignment.py:
  * set `LOCAL_TRAIN_FILE` / `LOCAL_TEST_FILE` to paths of the pickled CIFAR
    batches on your machine (autograder uses `../data/train`, `../data/test`).
  * implement `train`, `test` and fill in missing code in model classes.
  * run:

```bash
python assignment.py
```

The script will invoke `main()`; it uses `matplotlib` to pop up loss/accuracy
plots and expects TensorFlow 2.x (the code imports `tensorflow as tf`).
