**Purpose**

This repository is an assignment scaffold for building a TCP-like reliable data transfer protocol on top of UDP. The spec in README.md defines the required features: a custom packet header, 3-way handshake, 4-way close, checksum verification, retransmission, chunking, pipelining, flow control, and TCP Reno-style congestion control. The code in Header.py provides the packet format and byte serialization logic, while RDT.py is only a skeleton interface for the socket you are expected to implement.

It is not a complete working project as checked in. The core methods in RDT.py all raise `NotImplementedError`, and the harnesses in test_case.py, calculate_throughput.py, and calculate_latency.py still contain TODOs that must be adapted to your implementation. The only runnable finished component in the repo is the local proxy simulator in proxy.py.

**How To Run**

The intended workflow is:

1. Implement the reliable socket API in RDT.py.
2. Use Header.py to encode and decode packets with the required 42-byte header plus payload.
3. Update the TODO sections in test_case.py, calculate_throughput.py, and calculate_latency.py so they match your socket design.
4. For local testing, start the proxy with `python proxy.py`.
5. Run correctness testing with `python test_case.py`.
6. Run local performance checks with `python calculate_throughput.py` and `python calculate_latency.py`.

From the README, the expected environment is Python 3.9.0, standard library only, and a max payload size of 256 bytes per packet when using the provided proxy. For official testing, README.md and the commented blocks in test_case.py show the remote proxy addresses; for local testing, the scripts use `127.0.0.1`.
