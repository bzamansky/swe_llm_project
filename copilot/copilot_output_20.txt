**Project Intent**

This repository is a course project scaffold for building a reliable data transfer protocol on top of UDP, effectively a simplified TCP-like transport implemented at the application layer. The assignment specification is in README.md, and the required socket API is stubbed in RDT.py.

The intended implementation includes:

1. Connection setup with a 3-way handshake.
2. Reliable delivery with checksum verification and retransmission.
3. Data segmentation into payload chunks of at most 256 bytes.
4. Pipelined sending.
5. Flow control and congestion control modeled after TCP Reno.
6. Connection teardown with a 4-way handshake.

The protocol packet format is defined by Header.py and described in README.md. The required socket methods are:
RDTSocket in RDT.py

**How It Should Be Run**

The intended workflow is:

1. Implement the protocol in RDT.py.
2. Use the local proxy in proxy.py for local testing.
3. Run the provided test scripts.

Documented commands are in README.md:
```sh
python test_case.py
python calculate_throughput.py
python calculate_latency.py
```

What each script is for:

- proxy.py
  - Local unreliable-network simulator.
  - Opens UDP forwarding sockets and a small TCP result server.
  - Used by test_case.py.

- test_case.py
  - End-to-end correctness test harness against the proxy.
  - Spawns sender and receiver in separate processes.
  - Uses local addresses by default.
  - Expects your `RDTSocket` to fill in source/target addresses and `test_case` in headers.

- calculate_throughput.py
  - Compares plain UDP file transfer with your RDT file transfer locally.
  - Uses `original.txt` and writes `transmit.txt`.
  - Still requires you to fill in the RDT sender/receiver portions.

- calculate_latency.py
  - Measures one-way message latency for UDP and your RDT socket locally.
  - Also still assumes your `RDTSocket` is implemented.

Practical run order for local testing:

1. Start proxy.py in one terminal.
2. Run test_case.py in another terminal.
3. After the core socket works, run calculate_throughput.py and calculate_latency.py.

Notes from the spec that affect running it:

- Python version expected by the assignment: README.md
- Payload per packet must stay within 256 bytes because the full packet limit at the proxy is 298 bytes: README.md
- For official course testing, traffic is supposed to go through the provided remote proxy server addresses in README.md
- The local proxy/test addresses are described in README.md and mirrored in test_case.py
