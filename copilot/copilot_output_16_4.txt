**Project Intent**

This repository is a CS61C assignment scaffold for building `numc`, a Python extension module implemented in C that exposes a matrix type and accelerates matrix operations.

The intended architecture is:

1. A low-level C matrix library in matrix.h and matrix.c.
2. A Python-facing wrapper type `numc.Matrix` in numc.h and numc.c.
3. C unit tests in mat_test.c.
4. Python correctness/performance tests in unittests.py using a reference implementation called `dumbpy`.

A key detail: this repo is still a template or partially completed starter. Many core functions are still `TODO`, especially in matrix.c and numc.c, so the project is intended to implement these features rather than already providing a finished package.

**What It Is Supposed To Do**

The public object is `numc.Matrix`, registered in numc.c. From the constructor and method stubs, the intended Python API includes:

1. Creating matrices from dimensions:
   `Matrix(rows, cols)`
   `Matrix(rows, cols, fill_value)`

2. Creating matrices from Python lists:
   `Matrix(rows, cols, one_dimensional_list)`
   `Matrix(two_dimensional_list)`

3. Creating random matrices:
   `Matrix(rows, cols, rand=True, seed=..., low=..., high=...)`

4. Accessing metadata:
   `.shape`

5. Converting back to Python lists:
   `Matrix.to_list(matrix)`

6. Element access and mutation:
   `.get(row, col)`
   `.set(row, col, value)`
   `matrix[row, col]`
   slicing/indexing via the mapping protocol

7. Operator overloads:
   `+`, `-`, unary `-`, `abs(matrix)`, `*` for matrix multiplication, and `**` for matrix power

Those operations are declared in numc.h and wired into Python type hooks in numc.c, while the underlying C functions are declared in matrix.h.
