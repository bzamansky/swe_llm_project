**Project Intent**
This repository is a CS61C lab skeleton for learning parallelism and basic systems programming in C.

- The OpenMP side benchmarks vector addition and dot product implementations across different thread counts in v_add.c, dotp.c, and hello.c.
- The web server side serves static files from files, exposes a compute endpoint that runs the dot-product benchmark, and exposes an image-processing endpoint that applies a Sobel filter to BMP images. The server entry point is server.c, with most logic in server_utils.c and compute/image code in omp_apps.c.
- The code is not fully finished in this checkout. There are explicit student TODOs in omp_apps.c and unfinished process-handling code in server_utils.c, so this is intended as an assignment/lab, not a completed production project.

**What Each Program Does**
- hello.c: minimal OpenMP ‚Äúhello world‚Äù that prints one line per thread.
- v_add.c: allocates large vectors, runs vector-add implementations repeatedly, sweeps thread counts from `1` to `omp_get_max_threads()`, verifies correctness, and prints timing results.
- dotp.c: does the same for dot product, comparing naive, manual-reduction, and reduction-based implementations.
- server.c: parses CLI flags, changes into the served file directory, and starts the HTTP server.
- server_utils.c: request parsing/dispatch, static file serving, directory listings, `/report`, and `/filter/...bmp`.
- omp_apps.c: shared compute kernels plus the Sobel image filter.
- libhttp.c: tiny local HTTP parsing/response library.
- libbmp.c: tiny local BMP read/write library.
