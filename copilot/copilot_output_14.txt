**Purpose**

This repository is a small notebook-based assignment for the early “collecting data” phase of a machine learning project. The notebook asks the student to do two things:

1. Define at least five topics of interest, each with a 50+ word explanation, in what_are_topics_you_are_interested_in() in collecting_data.ipynb.
2. For at least three of those topics, list at least three candidate dataset sources in find_some_datasets() in collecting_data.ipynb.

The surrounding markdown in collecting_data.ipynb makes the intent explicit: explore possible ML project areas, identify what data might exist, and think about dataset quality and bias before moving on. The grading criteria in rubric.md and the tests in test_hw.py enforce those minimum requirements.

**How To Run It**

The setup and run path is in readme.md. From the repository root:

python -m venv ./.collecting-data

On macOS or Linux:

source ./.collecting-data/bin/activate

Then install dependencies from requirements.txt and run:

pytest

The README also says the intended reading order is rubric.md first, then collecting_data.ipynb. The tests work by importing code directly out of the notebook through the custom loader in test_utils.py, so this is not a conventional package or app entrypoint.

**APIs, Commands, Libraries**
