**Overview**

This repository is a local copy of the Berkeley/edX CS188 Pacman assignments. It is not one single application; it is three mostly self-contained teaching projects built on the same Pacman framework: search, adversarial search, and reinforcement learning. The top-level intent is stated in README.md, and each project declares the student-editable files in its own projectParams.py, projectParams.py, and projectParams.py.

The three projects are:

1. Project 1: generic search in Pacman. Students implement DFS, BFS, UCS, A*, and heuristics in search.py and use them through agents defined in searchAgents.py.
2. Project 2: multi-agent game search. Students implement reflex evaluation, minimax, alpha-beta pruning, expectimax, and a stronger state evaluator in multiAgents.py.
3. Project 3: reinforcement learning and MDPs. Students implement value iteration, Q-learning, approximate Q-learning, and analysis answers in valueIterationAgents.py, qlearningAgents.py, and analysis.py.

**How To Run**

Run each project from inside its own folder. That matters because each project expects local relative paths like layouts and test_cases, and the projects duplicate filenames such as pacman.py, util.py, and game.py.

Environment notes:

1. The codebase is written in Python 2 style, not modern Python 3. Evidence includes print statements without parentheses, Tkinter with a capital T, and cStringIO in graphicsUtils.py, util.py, and the matching files in Projects 2 and 3.
2. For graphical mode, Tkinter must be available.
3. There is no requirements.txt or package manifest; this repo relies on the Python standard library plus Tkinter.

Common run patterns:

1. Project 1:
   cd "Project 1 - Search in Pacman"
   python pacman.py
   python pacman.py -l tinyMaze -p SearchAgent -a fn=bfs
   python autograder.py
   python autograder.py -q q1 --no-graphics
   python eightpuzzle.py

   The concrete example commands are listed in commands.txt.

2. Project 2:
   cd "Project 2 - Multi-Agent Pacman"
   python pacman.py
   python pacman.py -l smallClassic -z 2
   python pacman.py -p ReflexAgent
   python pacman.py -p MinimaxAgent
   python pacman.py -p AlphaBetaAgent
   python pacman.py -p ExpectimaxAgent
   python autograder.py
   python autograder.py -q q2 --no-graphics

   The Pacman CLI usage is built into pacman.py, and the autograder options are in autograder.py.

3. Project 3:
   cd "Project 3 - Reinforcement Learning"
   python gridworld.py
   python gridworld.py -a value -i 100
   python gridworld.py -a q -k 100
   python pacman.py -p PacmanQAgent -a epsilon=0.1
   python analysis.py
   python autograder.py
   python autograder.py -q q1 --no-graphics

   The gridworld options are defined in gridworld.py, the Pacman RL command-line interface is in pacman.py, and the Q-learning argument example is in qlearningAgents.py.

Useful command-line flags used across the Pacman entry points:

1. -l or --layout to choose a map.
2. -p or --pacman to choose the Pacman agent class.
3. -a or --agentArgs to pass agent parameters such as fn=bfs or epsilon=0.1.
4. -t for text mode and -q for quiet text mode.
5. -n for number of games.
6. -x for training episodes.
7. --no-graphics on the autograders.

One repo inconsistency is worth noting: README.md says useful commands can be found under Project xx/commands.txt, but this checkout only has that file in Project 1.
