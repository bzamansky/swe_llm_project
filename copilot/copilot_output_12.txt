**Purpose**

This repository is a student ETL lab for a fictional retailer, TechMart. Its intended goal is to extract data from four source systems, clean and validate it, and load it into a SQL Server star-schema data warehouse for analytics.

The clearest sources for intent are:
- etl_lab_assignment.md
- etl_pipeline.py
- data_warehouse_schema.sql

What it is designed to do:
- Read sales transactions from a CSV file
- Read product catalog data from a JSON file
- Read customer profiles from MongoDB
- Read support ticket data from SQL Server
- Validate data quality across all sources
- Transform raw source data into warehouse-ready dimensions and facts
- Load a TechMartDW warehouse with SCD Type 1 style dimension handling
- Generate operational, quality, and business metrics for monitoring and Power BI-style reporting

The target warehouse defined in data_warehouse_schema.sql contains:
- Dimensions:
  - dim_date
  - dim_customers
  - dim_products
  - dim_stores
  - dim_support_categories
- Facts:
  - fact_sales
  - fact_customer_support
- Metadata tables:
  - etl_pipeline_runs
  - data_quality_issues

One important caveat: this is intentionally a partially completed lab, not a finished product. The assignment and tests show that some functions and many tests still contain TODOs, especially in:
- transformers.py
- test_unit.py
- test_integration.py
- test_e2e.py

So the repository is intended to run end to end, but in its current state some parts are scaffolding for students to complete.
