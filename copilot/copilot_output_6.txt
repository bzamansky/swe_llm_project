The repository is **not** a finished application – it’s the skeleton for the
“Advanced‑Python‑Techniques” project that you would complete as part of the
Udacity course.  Its goal is to give you a small command‑line program that
loads two real NASA datasets (one CSV and one JSON), turns each row/record into
Python objects, links them together, and then lets you **inspect** individual
asteroids/comets or **query** the close‑approach data with a variety of
filters.  You can also run an **interactive REPL** so that you can issue
multiple queries without re‑reading the files each time.

---

## What the code is meant to do

* read neos.csv (NEO properties) and cad.json (close–approach
  records)  
* convert rows into instances of two classes in models.py:
  * `NearEarthObject` – designation, (optional) name, diameter, hazardous flag,
    plus a list of linked approaches
  * `CloseApproach` – time, distance (au), velocity (km s⁻¹), and a reference
    back to its NEO  
  (helper functions in helpers.py translate the NASA date strings)
* build an `NEODatabase` in database.py that
  * keeps lists of all NEOs and approaches
  * maintains lookup tables for fast `get_neo_by_designation` /  
    `get_neo_by_name`
  * scans through the approaches and “links” each one to its NEO
  * implements `query(filters)` that yields approaches matching every
    predicate
* construct filter objects in filters.py from command‑line options
  (`date`, `min‑distance`, `hazardous`, …) and a `limit` helper
* emit results to stdout or to a file (`write_to_csv` / `write_to_json`)

The main.py script is the CLI entry point; it parses arguments, calls the
extractors, builds the database, and dispatches to one of three sub‑commands.

---

## How to run it

```sh
# show help
python3 main.py --help

# inspect a single object
python3 main.py inspect --pdes 433
python3 main.py inspect --name Halley --verbose

# query the close‑approach table
python3 main.py query --date 1969-07-29 --min-distance 0.1 --limit 5
python3 main.py query --start-date 2020-01-01 --outfile results.json

# interactive REPL (keeps data in memory)
python3 main.py interactive
# use `i`/`q` shorthand and `help` inside the prompt
```

The default data files live under data but may be overridden with
`--neofile`/`--cadfile`.  Output files must end in `.csv` or `.json`; the code
infers the format from the extension.

## APIs, commands and libraries used

* **Standard‑library modules only** – there are no external dependencies.
  * `argparse` – parsing command‑line options and sub‑commands
  * `cmd` – simple REPL for `interactive`
  * `csv`, `json` – reading/writing the datasets and results
  * `datetime` – parsing and formatting dates
  * `pathlib` – file‑path handling
  * `shlex` – splitting shell‑like strings inside the REPL
  * `operator` – comparator functions for filter objects
  * `sys`, `time` – misc. utilities (stderr, timestamping)
* The project doesn’t call any web APIs itself; the JSON/CSV files were
  downloaded from NASA/JPL’s CNEOS APIs (links are in the README).
* No third‑party libraries such as `requests`, `pandas`, etc. are present.
