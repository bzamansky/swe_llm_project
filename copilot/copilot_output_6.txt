**Summary**

This repository is a starter project for a command-line Python application that explores near-Earth object data from NASA/JPL.

Its intended job is to:

1. Load NEO metadata from a CSV file and close-approach records from a JSON file.
2. Convert those raw records into Python objects.
3. Link each close approach to its corresponding NEO.
4. Support lookup of a single NEO by designation or name.
5. Support filtered queries over close approaches by date, distance, velocity, diameter, and hazard status.
6. Print results to the terminal or write them to CSV or JSON.

The design and examples are described in README.md. The CLI entry point is main.py. The implementation is spread across models.py, extract.py, database.py, filters.py, helpers.py, and write.py.

**Important status note**

This checkout is not a finished application yet. The core modules still contain TODOs, so the repository is primarily an instructional starter template. The intended behavior is well defined by the README and tests, but the current code will not fully perform that behavior until those TODOs are implemented.

**What It Is Intended To Do**

The program works with two provided datasets under the data folder:

1. neos.csv
   Contains near-Earth object properties such as:
   - primary designation
   - optional human-readable name
   - diameter
   - potentially hazardous flag

2. cad.json
   Contains close approach events such as:
   - approach date/time
   - distance from Earth in AU
   - relative velocity in km/s
   - designation of the approaching object

From that data, the finished tool should let you:

1. Inspect a single NEO
   - by primary designation
   - by name
   - optionally list all known close approaches for that object

2. Query close approaches
   - on a specific date
   - within a date range
   - by min/max distance
   - by min/max velocity
   - by min/max NEO diameter
   - by hazardous or not hazardous status

3. Save query results
   - to CSV
   - to JSON

4. Use an interactive shell
   - run repeated inspect/query commands without reloading the dataset each time
