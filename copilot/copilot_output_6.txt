This repository is a course starter project for exploring near-Earth object data from NASA/JPL. The intended end state is a command-line tool that loads two local datasets, links NEO records to close-approach records, lets you inspect a single object, query approaches with date/distance/velocity/diameter/hazard filters, and optionally export results as CSV or JSON. The design is laid out in README.md and wired through main.py, but the core implementation files still contain TODOs, so the repository as checked in is instructional scaffolding rather than a finished application.

You’re meant to run it with Python 3.6+ from the project root. The main entrypoint is:
python3 main.py --help

The supported subcommands are:
inspect
query
interactive

Examples described by the repo:
python3 main.py inspect --pdes 433
python3 main.py inspect --name Halley
python3 main.py query --date 2020-03-14 --max-velocity 25 --min-diameter 0.5 --hazardous
python3 main.py query --limit 5 --outfile results.csv
python3 main.py interactive

For validation, the repo uses the standard library unittest suite:
python3 -m unittest --verbose

The project uses local data files in the data folder, not live network calls at runtime. The APIs referenced are NASA/JPL data sources that produced those files:
1. NASA/JPL SBDB small-body data, represented locally by neos.csv.
2. NASA/JPL CNEOS Close Approach Data API, represented locally by cad.json.
3. The README also points to NASA’s SBDB and CAD documentation pages for schema/background, but the code itself does not call those APIs directly.

Libraries are almost entirely Python standard library. The application code uses argparse, cmd, csv, json, datetime, pathlib, shlex, sys, time, and operator. The tests use unittest and a few more standard modules like math, io, contextlib, collections, and unittest.mock. There are no third-party Python dependencies and no requirements.txt or pyproject.toml in this repo.

At a module level, the intended architecture is:
main.py: CLI parsing and command dispatch
extract.py: load CSV/JSON into objects
models.py: NearEarthObject and CloseApproach models
database.py: cross-link objects and support lookup/query
filters.py: build query predicates and result limiting
write.py: export query results to CSV/JSON
helpers.py: date parsing/formatting helpers
