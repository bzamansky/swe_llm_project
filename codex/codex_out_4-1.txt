Project Intent

This repository is a Python 3.12 homework scaffold for nanocli, a minimal terminal-native coding agent. Its intended job is to talk to the OpenAI Responses API, maintain a conversation context, let the model request local tools, execute those tools, feed tool results back to the model, and automatically compact long conversation history when token usage crosses a configured threshold.

The assignment focus is HW4: automatic context compaction. The agent should detect when usage.total_tokens from the most recent completed Responses API call reaches --auto-compact-token-limit, ask the model to summarize the old context using a fixed fallback prompt, replace the old context with a synthetic checkpoint message, and then keep working from that summary.

Important note: the repo is still a starter scaffold. Several required functions currently raise NotImplementedError, especially in 
src/nanocli/core.py
 and 
src/nanocli/tool.py
. The tests define the expected behavior.

How To Run

Set up and run everything through uv:

uv run pytest
Run the CLI:

uv run nanocli --model gpt-5.4
Useful CLI options from 
src/nanocli/cli.py
:

uv run nanocli --model MODEL_NAME
uv run nanocli --model MODEL_NAME --yolo
uv run nanocli --model MODEL_NAME --instructions-path configs/tool_use_instructions.txt
uv run nanocli --model MODEL_NAME --instructions-as-system
uv run nanocli --model MODEL_NAME --reasoning_effort medium
uv run nanocli --model MODEL_NAME --auto-compact-token-limit 320
Inside the interactive CLI:

/debug
prints the current session configuration, instructions, compaction prompt, and stored context.

Run the local mock Responses server:

uv run nanocli-mock
By default it listens at:

http://127.0.0.1:8000/v1
The handout says these should be health checks:

uv run nanocli-mock
uv run nanocli --model nothing
Run formatting/lint/type checks with prek:

uv tool install prek
prek run --all-files
The homework deliverable commands are:

uv run pytest -v --tb=no > test_log.txt
prek run --all-files > prek_log.txt
Main Components


src/nanocli/cli.py
 is the terminal interface. It parses CLI flags, loads the instruction prompt, creates a Session, reads user input, prints assistant messages, displays tool calls/results, asks for approval unless --yolo is enabled, and shows a compaction notice when session.compaction_count increases.


src/nanocli/core.py
 is the agent loop and data model. It defines Message, Reasoning, ToolCallInput, ToolCallOutput, ToolApprovalRequest, and Session. It calls the Responses API through OpenAI().responses.create(...), streams until response.completed, converts API output items into local context items, tracks usage.total_tokens, and is supposed to implement maybe_compact_session(...) plus run_turn(...).


src/nanocli/tool.py
 defines the local tool protocol and registry. It registers four tools: bash, read_file, list_dir, and edit_file. bash is implemented; the file/directory/edit tools are currently TODOs.


src/nanocli/_dev/mock_server.py
 is a local Flask mock of the OpenAI Responses API. Tests use it to simulate streaming responses, tool calls, token usage, and long-horizon compaction behavior.


configs/tool_use_instructions.txt
 is the default instruction prompt: “You are NanoCLI, a minimal, precise command-line code agent...”


hw4.md
 is the assignment spec.

APIs Used

The main external API is the OpenAI Responses API via the openai Python package:

session.client.responses.create(
    model=session.model,
    input=input_items,
    tools=tool_specs(),
    tool_choice="auto",
    include=["reasoning.encrypted_content"],
    stream=True,
    store=False,
    prompt_cache_key=session.prompt_cache_key,
    ...
)

Expected Behavior From Tests

The tests expect the final implementation to:

Send strict tool schemas for bash, read_file, list_dir, and edit_file.
Ask for approval before running non-read-only tools unless --yolo is set.
Execute all tool calls in a model response, including multiple calls in one response.
Send function_call_output items back into the next Responses API request.
Preserve tool errors as model-visible outputs.
Support instructions either as the Responses API instructions parameter or as a first system input when --instructions-as-system is used.
Track completed response usage.total_tokens.
Compact context only when last_token_usage_total_tokens >= auto_compact_token_limit.
Append the fixed SUMMARIZATION_PROMPT as a synthetic user message during compaction.
Replace old history with one checkpoint user message using SUMMARY_TEMPLATE.
Reset token tracking after compaction and increment compaction_count.
Continue long-running mock tasks after compaction.
