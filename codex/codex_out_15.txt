This repository is a starter handout for a Python homework called “Housing Priority Calculator.” The intended project is a small command-line app where a student answers housing-related questions, and the program computes a housing priority score. The design is intentionally split into three parts: src/question_asker.py, which gathers validated input; src/priority_calculator.py, which assigns points; and src/hw1.py, which orchestrates the flow. The assignment is also teaching test-first development: students are supposed to define a scoring system in Summary.md, write tests first, then implement the skeleton methods.

To run it, install the small tool dependencies with pip install -r requirements.txt, then use python3 test_runner.py to run all tests or python3 test_runner.py -t <test_file> to run selected tests. Coverage is generated with ./generate_coverage_reports.sh, and the intended manual app entrypoint is python3 hw1.py per the README, though in this handout the real file currently lives at src/hw1.py. The custom test harness in test_runner.py uses unittest discovery to run both test_*.py and impl_*.py files under tests/.

There are no external product APIs used by the student code itself. The code only uses Python’s standard library APIs like input(), print(), argparse, unittest, and unittest.mock.patch. The only non-stdlib libraries listed in requirements.txt are coverage and pylint, which support testing and linting rather than app logic. The GitHub workflow in .github/workflows/grade.yml submits to Pawtograder via the pawtograder/assignment-action, so that is the only external service integration present in the repo.
