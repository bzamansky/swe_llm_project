What It Is

This repo is a homework starter for building a small terminal coding agent called nanocli that uses the OpenAI Responses API’s structured tool-calling interface instead of parsing bash blocks out of plain text. The intended flow is: send user input to the model, let the model emit function_call items, execute those tools locally, send back function_call_output, and repeat until the model stops asking for tools. That design is described in hw2.md, and the implementation skeleton lives in core.py (line 73) and tool.py (line 72).

The CLI itself is intentionally thin: it just parses args, starts a Session, prints assistant/tool output, and asks for approval before non-read-only tools unless --yolo is used, as shown in cli.py (line 84). The project is not finished yet: read_file, list_dir, edit_file, and the main run_turn loop are still NotImplemented, and uv run pytest currently fails with 23 failing tests. You can see the missing TODOs in tool.py (line 223) and core.py (line 168).

How To Run It

Use uv for everything. The README says to run tests with uv run pytest and lint/hooks with prek run --all-files in README.md (line 17). The package exposes two commands in pyproject.toml (line 11):

uv run nanocli --model <model-name>
uv run nanocli-mock
Useful runtime flags from cli.py (line 130):

--yolo to skip approval prompts
--instructions-path to swap the system prompt file
--instructions-as-system to send instructions as a system message instead of the instructions parameter
--reasoning_effort with none|minimal|low|medium|high|xhigh
For real model calls, the app constructs an OpenAI() client in core.py (line 84), so in practice it expects normal OpenAI client configuration such as an API key. For local testing, the mock server in mock_server.py (line 293) simulates Responses API behavior with triggers like mock:single, mock:multi, mock:read, and mock:edit.
