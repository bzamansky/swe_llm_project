What This Repo Is

This repository is a starter/final-project scaffold for implementing a TCP-like reliable data transfer protocol on top of UDP in Python. The assignment expects students to build an RDTSocket that supports:

connection setup with a 3-way handshake
teardown with a 4-way handshake
checksum validation
retransmission on loss/corruption/delay
segmentation and reassembly of larger payloads
pipelining
flow control and congestion control
That intent is described in README.md (line 1) and reflected in the required socket methods in RDT.py (line 3).

One important caveat: this checkout is not a finished implementation. RDTSocket is still a template with NotImplementedError in every major method in RDT.py (line 17), and the test harness scripts also still contain TODOs in test_case.py (line 126), calculate_throughput.py (line 90), and calculate_latency.py (line 80).

What I changed

In RDT.py, I turned RDTSocket into a minimal working UDP-backed wrapper using the existing RDTHeader API from Header.py. It now:

creates and binds a UDP socket
supports a lightweight accept() / connect() handshake
serializes data with RDTHeader.to_bytes()
receives packets with RDTHeader.from_bytes()
sends a simple FIN/ACK close signal
tolerates socket.timeout and BlockingIOError
enables SO_REUSEADDR so repeated local runs don’t fail on reused ports
In test_case.py, I only filled the existing TODOs:
