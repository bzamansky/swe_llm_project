This repository is a TensorFlow homework scaffold for building and training image classifiers on a 2-class subset of CIFAR-10, specifically cats vs dogs. The intended workflow is spelled out in assignment.py (line 112): load CIFAR-style pickled train/test data, initialize either an MLP or CNN, train for roughly 10-25 epochs, and reach at least 70% test accuracy on the cat/dog subset. The repo also includes a separate manual convolution exercise: manual_convolution.py (line 11) defines a custom ManualConv2d layer, and local_test.py (line 243) compares it against Keras Conv2D.

How it should be run, based on the code:

Main training entrypoint is python assignment.py, after filling in the TODOs and pointing LOCAL_TRAIN_FILE / LOCAL_TEST_FILE or using the autograder paths in assignment.py (line 126).
Local sanity tests for the custom convolution are run with python local_test.py; that file directly executes several tests in its __main__ block in local_test.py (line 243).
The preprocessing path is get_data(file_path, classes) and batching path is get_next_batch(...) in preprocess.py (line 23).
Right now the repo is not runnable end-to-end as-is: train, test, main, CifarModel.loss, CifarModel.accuracy, MLP, CNN, and get_data are still incomplete in assignment.py (line 18), base_model.py (line 9), mlp.py (line 16), cnn.py (line 17), and preprocess.py (line 31).
APIs, commands, and libraries used:

Libraries: tensorflow, keras, numpy, matplotlib, pickle, and Python stdlib modules os, math, random, abc.
TensorFlow / Keras APIs used or expected: tf.keras.Model, tf.keras.layers.Layer, tf.Variable, tf.random.truncated_normal, tf.one_hot (mentioned in docstring), tf.gather, tf.image.random_flip_left_right, tf.keras.initializers.Constant, and keras.layers.Conv2D.
File/data format: CIFAR pickled batches loaded with pickle.load(...) in preprocess.py (line 7).
Commands you’d realistically use here: python assignment.py for training and python local_test.py for convolution validation.
There are no external network APIs or web services in this repo; “APIs” here are library APIs only.
